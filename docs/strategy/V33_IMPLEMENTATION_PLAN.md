## 一、对 V3.3 修订的简要评价

- **已闭合的争议点**：容错机制、次日 S 的实盘定义（S_intra）、趋势过滤与市场/个股口径、同一事件定义、1 小时涨幅的日频近似、席位前 10 字、归一化两步法、ADX 连续 3 日日环比、新策略仓位 40% 含义、信号生效时点表、情感一期词典/SnowNLP、席位无标签默认 1.0、人工标注覆盖等，均已写清，**可直接作为开发规格**。
- **唯一建议再补一句**（非阻塞）：2.1.2 卖出信号里，“次日 S 回落”在实盘是否也采用与买入对称的 **T+1 盘中 S_intra < T 日 S** 的估算规则？若与买入一致，可在该段加一句：“实盘定义同买入，采用 T+1 盘中 S_intra 与 T 日 S 比较，若 S_intra < T 日 S 则视为回落。”避免实现时歧义。
- **数据源占位符**：文档中“[指定数据商]”“Wind（.WI）”等，落地时在配置或数据字典中统一替换为实际数据源（如 akshare/ tushare/ Wind 等），便于后续切换。

## 三、Phase 0：数据与配置底座（建议优先）

**目标**：为情绪、消息、政策、龙虎榜提供统一的数据接口与配置，避免各策略各自拉数据、口径不一致。

| 序号 | 任务 | 说明 | 产出物 |
|------|------|------|--------|
| 0.1 | 数据源配置表 | 列出各策略所需指标及对应数据源（含占位 [指定数据商]）、更新频率、是否实时 | `config/data_sources.yaml` 或文档 |
| 0.2 | 情绪指标数据接口 | 涨跌家数比、换手率、融资买入占比、HV20（沪深300）、新高新低比；期权 PCR 可选，缺则权重 0 | `src/data/sentiment/` 扩展，支持多指标与缺项归一化 |
| 0.3 | 行业指数约定 | 申万二级行业指数代码与数据源（或占位），供消息/政策“板块涨幅”“行业市值占比”使用 | 配置或 `src/data/industry.py` 占位 |
| 0.4 | 席位与新闻源配置 | 席位别名映射表（空表可先建）、新闻源权重（公告 1.5/主流 1.2/自媒体 0.5） | `config/seat_alias.csv`、`config/news_source_weights.yaml` |
| 0.5 | 交易成本与延迟配置 | 滑点、佣金、印花税、5 分钟延迟、T+1 次日 15 分钟等 | `config/trading_costs.yaml` 或沿用现有 |

**验收**：配置与接口就绪；情绪可仅用 2～3 个可得指标跑通 Z-score 与权重归一化（缺项权重 0）。

## 四、Phase 1：情绪因子 V3.3

**目标**：完全按 2.1 实现市场情绪指数与信号规则（含趋势过滤），并写清 S_intra 的估算方式。

| 序号 | 任务 | 说明 | 产出物 |
|------|------|------|--------|
| 1.1 | 多指标 Z-score 与合成 | 60 日窗口；反向指标取负；缺项权重 0 并归一化；初始权重按文档表 | `src/data/sentiment/sentiment_index.py` |
| 1.2 | S_low / S_high | 60 日 20/80 分位，每日收盘后重估 | 同上 |
| 1.3 | 信号规则（回测） | T 日 S < S_low 且 T+1 日 S > T 日 S → 买入；T 日 S > S_high 且 T+1 日 S < T 日 S → 卖出 | `src/strategies/sentiment.py` 重写 |
| 1.4 | 趋势过滤（个股） | MACD 柱 3 日线性回归斜率、5 日新高/新低、ADX(14)>25；仅当情绪极端+次日确认 与 趋势衰竭 同时满足才输出买卖 | 同上，需传入个股 df |
| 1.5 | S_intra 说明与实现（可选首版简化） | 技术实现文档：S_intra 用 T+1 已更新日频指标（融资、涨跌家数等）+ 历史权重估算；未更新指标用 T 日。若首版不做盘中，可先实现“T+1 收盘后确认、T+2 执行”并在文档注明 | `docs/strategy/SENTIMENT_TECH.md` + 可选 `sentiment_intraday.py` |
| 1.6 | 信号生效时点 | 情绪：T 日收盘计算，T+1 开盘执行（与 5.3 表一致） | 组合引擎或执行层 |

**验收**：回测可跑通；无未来函数；趋势过滤仅在同时满足情绪+个股条件时给信号。

## 五、Phase 2：消息面 V3.3

**目标**：按 2.2.1 实现新闻情感、24h N、预期差（日频近似）、新闻源权重与置信度。

| 序号 | 任务 | 说明 | 产出物 |
|------|------|------|--------|
| 2.1 | 情感打分与清洗 | 情感 [-1,1]；一期词典/SnowNLP+自定义词库；去重（同代码+同自然日+标题相似度>85% 或 同公告分类，取最早）；时间戳校验 | `src/data/news/` 扩展 |
| 2.2 | 新闻源权重 | 交易所 1.5、主流 1.2、自媒体 0.5；最终置信度 = min(基础置信度 × 新闻源权重, 1) | 配置 + 策略内使用 |
| 2.3 | 24h 同向 N 与 S_news | 以当前新闻时间向前 24h，同向最多 3 篇，加权平均 S_news | `src/strategies/news_sentiment.py` |
| 2.4 | 预期差（日频近似） | 盘中发布→当日收益率；盘后→次日收益率；申万二级行业指数涨幅>5% 时个股阈值放宽 | 同上 |
| 2.5 | 置信度公式 | 基础 = min(1, \|S_news\|×2 + 0.1×(N-1))；最终 = min(基础×新闻源权重, 1) | 同上 |
| 2.6 | 信号有效期 | 盘中至当日收盘，盘后至次日收盘；过期转持有 | 组合层 |

**验收**：单标的回测无未来函数；预期差用日收益替代 1 小时涨幅在文档与代码中一致标注。

## 六、Phase 3：政策面 V3.3

**目标**：按 2.3 实现政策识别、影响力、情绪强化、板块涨幅、重大利空与行业扩散。

| 序号 | 任务 | 说明 | 产出物 |
|------|------|------|--------|
| 3.1 | 关键词与影响力 | 附录 A 词库；利好/利空/中性/中性偏多/偏空；影响力 1.2/1.0/0.8/0.9/0.7 | `src/data/policy/` 扩展 |
| 3.2 | 买入条件 | 重大利好 + 情绪 S < S_high + 标的申万二级行业指数当日涨幅 < 2%；有效期 5 日 | `src/strategies/policy_event.py` |
| 3.3 | 置信度 | 影响力 × (1 + 情绪强化)；恐慌乘 1.3，贪婪乘 0.7 | 同上 |
| 3.4 | 重大利空 | 影响力≥1.0 + 关键词（监管加强、反垄断、集采降价、出口管制）+ 行业市值/全市场≥5%（申万二级、前一日收盘）；满足则无条件卖出 | 同上 + 行业市值数据接口 |
| 3.5 | 行业扩散 | 无行业关键词→全市场；有则仅对应申万二级；行业政策映射表可配置 | `config/policy_industry_mapping.csv` 或 yaml |
| 3.6 | 去重与时间 | 同一政策多渠道只保留一条，时间以政府官网为准；人工覆盖接口，修正次日生效 | 数据层 + 运维接口说明 |

**验收**：政策信号仅在发布时间后生效；重大利空优先在组合层可测。

## 七、Phase 4：龙虎榜/大宗 V3.3

**目标**：按 2.2.2 实现席位匹配、连续 2 日、占比与机构权重、大宗折价与买卖方、有效期。

| 序号 | 任务 | 说明 | 产出物 |
|------|------|------|--------|
| 4.1 | 席位匹配与别名 | 全称去后缀取前 10 字（或“证券”后部分）+ 别名表；无标签默认权重 1.0 | `src/data/money_flow/` + 配置 |
| 4.2 | 龙虎榜信号 | 连续 2 日同席位净买/卖、占比>0.1%、机构权重>1.2；置信度 = min(占比/0.2%,1)×席位权重均值；有效期 3 日 | `src/strategies/money_flow.py` |
| 4.3 | 大宗信号 | 折价<3%、买方长线机构、额>1 亿→买入(置信度 0.5)；折价>5%、卖方控股股东/董监高、额>0.5 亿→卖出(0.8)；有效期 5 日 | 同上 |
| 4.4 | 信号时点 | T 日交易 T+1 披露，信号 T+2 开盘执行；可选开盘后 15 分钟 | 组合/执行层 |

**验收**：席位匹配可复现文档示例；无标签席位不满足 >1.2。

## 八、Phase 5：组合与风控 V3.3

**目标**：投票池、信号有效期、动态权重（1/N、乘数、归一化两步法）、冲突与重大利空优先、新策略仓位 40%。

| 序号 | 任务 | 说明 | 产出物 |
|------|------|------|--------|
| 5.1 | 投票池与有效期 | 11 策略独立输出；消息/政策过期转持有；情绪当日无信号退出投票 | 组合引擎 |
| 5.2 | 基准权重与乘数 | 基准 1/11；震荡/趋势市乘数表；沪深300 的 ADX(14)、HV20 判定状态 | 同上 |
| 5.3 | 归一化两步法 | 先乘乘数再归一化；截断 [0.5%, 30%]；多出/不足按比例重分配；告警日志 | 同上 |
| 5.4 | 调整触发与冷却 | ADX/HV20 日环比连续 3 日超阈值则调整；调整后 7 日冷却 | 同上 |
| 5.5 | 冲突与重大利空 | 重大利空无条件卖出；否则买入总分 vs 卖出总分×1.2 | 同上 |
| 5.6 | 新策略仓位 40% | 情绪+消息+政策触发的开仓合计资金占比 ≤ 40%（硬上限） | 风控层 |

**验收**：权重和为 100%；截断与再分配可单测；重大利空优先可单测。

## 九、Phase 6：回测与实盘规范（贯穿各 Phase）

| 序号 | 任务 | 说明 |
|------|------|------|
| 6.1 | 未来函数约束 | 情绪用 T 及以前数据；新闻/政策从发布时间后生效；龙虎榜 T+2 开盘；在回测框架中显式校验 |
| 6.2 | 参数敏感性 | 情绪 20/80→[16,24]/[76,84]；新闻 0.3/-0.3→[0.24,0.36]/[-0.36,-0.24]；ADX 25→[20,30]；筛选：收益/回撤比变异系数最小 |
| 6.3 | 成本与延迟 | 滑点、佣金、印花税按文档；5 分钟延迟、盘后次日开盘、T+1 次日 15 分钟在回测与实盘一致 |
| 6.4 | 人工覆盖接口 | 政策标签修正入口与修正日志；修正次日生效、不追溯历史 |

## 十、建议实施顺序与时间线（示意）

```
Week 1–2   Phase 0 数据与配置底座
Week 2–4   Phase 1 情绪因子 V3.3（含趋势过滤；S_intra 可先文档后实现）
Week 4–5   Phase 2 消息面 V3.3
Week 5–6   Phase 3 政策面 V3.3
Week 6–7   Phase 4 龙虎榜/大宗（可与 Phase 2/3 部分并行）
Week 7–8   Phase 5 组合与风控
贯穿      Phase 6 回测规范、敏感性、成本与人工接口
```

- 若人力有限，可先做 **Phase 0 → Phase 1 → Phase 5（仅投票+有效期+冲突，不做动态权重）**，再补 Phase 2/3/4 和动态权重。
- 情绪与政策都依赖“市场情绪 S”：建议 Phase 1 先交付可用的 S（含缺项容错），再在 Phase 3 中复用 S 做情绪强化与 S < S_high 判断。

## 十一、与现有代码的衔接

- **现有**：`src/strategies/sentiment.py`、`news_sentiment.py`、`policy_event.py` 为简化版；情绪用单指标（沪深300 动量分位），消息/政策为关键词+简单阈值。
- **落地时**：  
  - 情绪：用 V3.3 多指标 Z-score 与权重表重写，并加趋势过滤（个股 MACD/5 日高低/ADX）。  
  - 消息：在现有新闻获取与情感基础上，补 24h N、新闻源权重、预期差日频近似与置信度公式。  
  - 政策：在现有关键词与影响力基础上，补板块涨幅（申万二级）、重大利空三条件与行业市值占比、行业扩散。  
  - 龙虎榜/大宗：新增模块，按 2.2.2 实现。  
  - 组合：在现有 `EnsembleStrategy` 或单独组合引擎中，增加信号有效期、1/N、乘数、归一化两步法、重大利空优先、新策略仓位 40%。

以上为 V3.3 落地计划；若你确认或微调阶段划分，可从 Phase 0 或 Phase 1 开始逐项在仓库里实现。
