# Alpha Factory 策略优化记录

## 2026-03-26: 双引擎调度架构（v5.0）🚀

### 核心升级：从单引擎加权到双引擎调度

**持仓分析优化**（v5.0.1）：
- **策略**：均值回归为主，趋势得分作为风险修正
- **基础判断**：使用均值回归引擎的score（超跌/超买逻辑）
- **风险修正**：
  - 趋势得分 < -0.3：降低加仓建议，提示"趋势转弱"
  - 趋势得分 > 0.3：提升持有信心，标注"趋势健康"
- **新增显示**：
  - 趋势得分 + 动量得分（可视化：📈强势 / ➡️中性 / 📉转弱）
  - 趋势预警信息（⚠️ 或 ✅）
- **分析理由**：增加趋势维度的说明，帮助理解操作建议

**问题背景**：
- v4.4应用优化权重后，发现回测数据天然偏向均值回归策略
- PB/PE/DUAL_REV等均值回归策略Sharpe 0.22-0.38，远超趋势策略0.02-0.08
- 导致推荐结果100%为超跌股，缺乏健康上涨标的
- **根本矛盾**：回测最优（震荡市数据）≠ 用户需求（多样化投资机会）

**解决方案**：架构升级为双引擎调度

#### 1. 新增趋势策略模块 (`src/strategies/trend_strategies.py`)

四个工程级趋势策略：
- **ADX_Trend**：趋势开关（基于ADX指标判断趋势强度）
- **MA_Alignment**：均线排列（多头/空头排列连续得分）
- **Momentum_Adj**：波动率调整动量（ATR标准化的动量因子）
- **Trend_Composite**：复合趋势因子（主力策略，融合ADX+均线+动量）

#### 2. 双引擎调度架构

**均值回归引擎**（原有策略）：
- 策略：PB、PE、PEPB、DUAL、BOLL、RSI、KDJ、MACD、MA
- 权重：MR_WEIGHTS（基于回测优化）
- 输出：超跌反弹型推荐

**趋势引擎**（新增）：
- 策略：Trend_Composite（复合趋势因子）+ Momentum_Adj（动量）
- 得分：trend_rank_score = 0.7×trend_score + 0.3×momentum_score
- 输出：趋势跟随型推荐

**软过滤机制**：
- 用趋势得分调整均值回归权重：adjusted_score = mr_score × (0.5 + 0.5×trend_weight)
- 避免硬剔除，增强自适应性

**市场状态感知**：
- 指数趋势强度：strength = (MA20 - MA60) / MA60
- 趋势市（strength > 3%）：超跌榜10只 + 趋势榜10只
- 震荡市（strength ≤ 3%）：超跌榜15只 + 趋势榜5只

#### 3. 推荐流程

```
全市场股票 (808只)
    ↓
[策略计算]
├── 均值回归引擎 → mr_score
└── 趋势引擎 → trend_score, momentum_score
    ↓
[软过滤]：adjusted_score = mr_score × (0.5 + 0.5×trend_weight)
    ↓
[市场状态判断]：指数MA20/MA60斜率 → 趋势市/震荡市
    ↓
[双榜单生成]
├── 超跌榜：按adjusted_score排序取N1
└── 趋势榜：按trend_rank_score排序，去重后取N2
    ↓
[合并推荐]：超跌榜 + 趋势榜，不足20只时按mr_score补全
    ↓
输出报告（终端 + MD文件）
```

#### 4. 配置参数

```python
# 双引擎调度配置
TREND_WEIGHT_BASE = 0.5       # 基础权重
TREND_WEIGHT_RANGE = 0.5      # 权重调整范围
TREND_SCORE_WEIGHT = 0.7      # 趋势得分权重
MOMENTUM_SCORE_WEIGHT = 0.3   # 动量得分权重
TREND_STRENGTH_THRESHOLD = 0.03  # 市场趋势强度阈值
```

#### 5. 技术实现

**文件修改**：
- 新增：`src/strategies/trend_strategies.py` (300行)
- 修改：`tools/analysis/recommend_today.py`
  - 添加趋势策略导入和配置 (1237-1258行)
  - 添加市场状态判断函数 (1260-1298行)
  - 修改 `run_full_12_analysis` 返回值，增加趋势得分 (1543-1598行)
  - 替换排序逻辑为双引擎调度 (1879-1928行)

**新增字段**：
- `trend_score`：趋势复合得分 [-1, 1]
- `momentum_score`：动量得分 [-1, 1]
- `trend_weight`：趋势权重 [0, 1]

#### 6. 预期效果

- **风格多样性**：超跌反弹 + 趋势跟随，满足不同风险偏好
- **市场自适应**：根据指数状态动态调整双榜单比例
- **回测最优保留**：均值回归引擎仍使用优化权重
- **用户体验提升**：不再100%推荐超跌股

#### 7. 后续优化方向

- 榜单比例：根据实际效果调整mr_n、trend_n

## 2026-03-26: 双轨推荐机制（v4.3）

### 核心改进：增加"健康上涨组"推荐 📈

**问题背景**：
- 用户反馈：所有推荐都是价格偏低或超跌的，没有推荐涨幅比较好比较健康的股票
- 根本原因：DUAL策略反向逻辑（将趋势跟随变为均值回归）导致系统偏好超跌标的

**解决方案**：采用**方案3+4组合**
- 保留现有TOP20推荐（超跌反弹型，DUAL反向逻辑）
- 新增"健康上涨组"推荐（趋势跟随型，独立筛选逻辑）

#### 健康上涨组筛选条件

1. **价格在MA20上方**：中期趋势向上
2. **价格在MA60下方50%以内**：不追高，有安全边际
3. **20日涨幅在3%-40%之间**：健康上涨，包括温和上涨
4. **5日涨幅>-2%**：短期未大幅回调
5. **量比>0.5**：有基本资金参与
6. **综合得分>3**：策略基本认可
7. **排除ST股票**

#### 实施效果

- **2026-03-26 实测**：从808只全市场股票中筛选出19只健康上涨标的
- **典型案例**：
  - 大唐发电（601991）：20日+14.14%，多头排列，MACD金叉
  - 江苏国信（002608）：20日+8.47%，多头排列，MA金叉
  - 大秦铁路（601006）：20日+6.71%，量价配合（量比1.2x）

#### 报告结构优化

```
1. 我的持仓
2. 今日持仓操作建议
3. 每日推荐 TOP20（超跌反弹型为主）
4. 健康上涨组 TOP10（趋势跟随型）⭐ 新增
5. 专题板块推荐（如核电+算电协同）
6. 历史推荐（最新在前）
```

#### 技术实现

- **文件**：`tools/analysis/recommend_today.py`
- **新增函数**：`_generate_healthy_growth_section(all_results, today)`
- **调用位置**：`save_incremental_report` 函数中，在专题板块推荐之前
- **数据复用**：使用已分析的 `all_results`，无需重复计算

#### 用户价值

- **风险分散**：提供超跌反弹和趋势跟随两种风格的标的
- **策略互补**：超跌型适合抄底，健康上涨型适合趋势跟随
- **真实反映市场**：当市场整体偏弱时，健康上涨组数量自然减少（如3月26日仅19只）

## 2026-03-24: L2层权重调整验证（v4.2）

### 核心发现：固定权重优于动态调整 🎯

经过严格的历史回测验证（30只股票，2023-2026年，1296个配置），**确认当前固定权重策略已是最优方案**。

#### 验证过程

**文件**: `tools/optimization/optimize_regime_weights.py`

1. **数据准备**：30只股票，800+ bars/股，覆盖牛熊震荡
2. **信号预计算**：逐bar运行12个子策略，记录18001个有效信号
3. **市场状态标注**：牛市4只、熊市4只、震荡22只
4. **网格搜索**：测试1296个权重配置（牛市/熊市 × 动量/价值 × 多组系数）
5. **时间切分验证**：训练期70%（2023-04-14 ~ 2025-05-06），测试期30%（2025-05-06 ~ 2026-03-19）

#### 关键结果

| 配置类型 | 测试期夏普 | 测试期年化收益 | 测试期最大回撤 | vs基线 |
|---------|-----------|---------------|---------------|--------|
| **基线（固定权重）** | **0.118** | **5.33%** | **5.98%** | --- |
| 最优动态配置 | 0.078 | 5.06% | 6.00% | ❌ -0.040 |
| TOP30平均 | 0.056 | 4.81% | 6.06% | ❌ -0.062 |

**结论**：所有30个最优动态配置在测试期均**显著劣于**固定权重基线。

#### 原因分析

1. **市场状态识别滞后**：MA20/MA200判断市场状态存在1-2个月滞后，错过最佳调整时机
2. **过拟合风险**：训练期最优系数在测试期失效（训练夏普-0.044 → 测试夏普0.078，仍不如基线0.118）
3. **系数叠加副作用**：即使限制单因子≤1.3x，组合后仍可能放大噪音而非信号
4. **固定权重已优化**：当前权重（BOLL 1.5, MACD 1.3等）已通过v3/v4多轮优化，难以进一步提升

#### 实施决策

- ✅ **保持L3层固定权重投票**（当前架构）
- ❌ **不启用L2层动态权重调整**（验证失败）
- ✅ **保留L2层代码框架**（未来可能有更好的市场状态识别方法）
- ✅ **新增系数上限规则**（`src/core/regime_adjustment_rules.py`）：
  - 单因子最大1.3x
  - 组合最大1.5x
  - 最小0.7x

#### 技术改进

1. **新增模块**：`src/core/regime_adjustment_rules.py`
   - 定义系数上限和叠加规则
   - 提供 `RegimeAdjustmentEngine` 类（备用）
   - 支持从优化结果加载系数

2. **优化工具完善**：
   - `optimize_regime_weights.py`：L2层系数优化（已运行，结论为不启用）
   - `optimize_thresholds.py`：信号阈值优化（已存在）
   - `calibrate_weights.py`：权重校准（已存在）

3. **Sentiment策略澄清**：
   - 当前设计**已正确**：市场级情绪数据（S/S_low/S_high）+ 个股趋势过滤（MACD/ADX）
   - CHANGELOG v4.1中"Sentiment调用错误"的描述有误，已更正理解

## 2026-03-22: DUAL反向 + 投票机制优化（v4）

### 改动内容

**文件**: `src/strategies/ensemble.py`

#### 1. DUAL信号反向处理（第400-416行）
- **原因**：IC分析显示DUAL策略与未来收益强负相关（IC=-0.3877）
- **验证**：反向后IC变为+0.3877（p<0.000001，统计显著）
- **实现**：BUY→SELL, SELL→BUY，confidence和position同步反向
- **效果**：将最差策略转化为最强预测因子

#### 2. 投票机制改进（第555-608行）
- **旧机制问题**：相对比例 `buy_pct = buy_score / total_active` 对权重不敏感
  - 例：3个策略投BUY，权重(1,1,1)和(10,1,1)的buy_pct都接近100%
  - 导致调整权重无法改变决策
  
- **新机制**：净得分 `net_score = (buy_score - sell_score) / active_weight_sum`
  - 权重高的策略对net_score贡献更大
  - 阈值校准为0.07（平衡交易频率和信号质量）
  - 最小票数降为1（避免过度保守，DUAL单票也有效）

#### 3. 保持原始权重（第147-159行）
```python
weights = {
    'BOLL': 1.5, 'MACD': 1.3, 'KDJ': 1.1, 'MA': 1.0,
    'DUAL': 0.9, 'RSI': 0.8, 'PEPB': 0.8, 'PE': 0.6,
    'PB': 0.6, 'NEWS': 0.5, 'MONEY_FLOW': 0.4,
}
```
