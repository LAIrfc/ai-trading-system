# Alpha Factory 策略优化记录

## 2026-03-23: 关键Bug修复（v4.1）

### 修复内容

**文件**: `tools/analysis/recommend_today.py`

#### 1. 致命缩进错误修复（第1588行）
- **问题**：`df = fetch_stock_data(code, 200)` 缩进错误，在循环外执行
- **后果**：所有808只股票共享同一份K线数据，价格/均线/量比全部相同
- **修复**：将该行缩进到循环内部，每只股票独立获取数据

#### 2. DUAL策略信号不一致修复（第1085-1101行）
- **问题**：`run_full_11_analysis` 中DUAL策略未反向，与 `ensemble.py` 不一致
- **后果**：full_11模式与ensemble模式的DUAL信号完全相反，无法对比验证
- **修复**：在 `run_full_11_analysis` 中添加DUAL反向逻辑（BUY↔SELL）

#### 3. 策略实例重复创建优化（第1054-1073行）
- **问题**：每个股票分析都重新创建11个策略实例（860次×11=9460次创建）
- **后果**：性能严重浪费，NewsSentiment/MoneyFlow涉及LLM初始化，耗时极长
- **修复**：使用全局单例模式 `_get_global_strategies()`，策略实例只创建一次

#### 4. 基本面数据验证改进（第1140-1250行）
- **问题**：PE/PB数据缺失时仍尝试分析，浪费计算资源
- **修复**：提前验证 `has_pe_data` / `has_pb_data` / `has_pepb_data`，数据不足时跳过

#### 5. 架构升级：分层决策框架（ensemble.py 第129-420行，recommend_today.py 第1062-1280行）
- **问题**：`run_full_11_analysis` 与 `ensemble.py` 策略集合不一致
  - 缺少 PEPB（权重0.8）
  - Sentiment 和 PolicyEvent 未纳入 ensemble
  - NEWS命名不一致（NewsSentiment vs NEWS）
- **深度分析**：
  - **SENTIMENT**：虽基于市场级情绪指标（S/S_low/S_high），但有**个股趋势过滤**（MACD柱斜率、ADX、5日新高/新低，满足2/3才触发），能产生差异化信号
  - **POLICY**：纯市场级政策新闻，所有股票信号完全相同，不适合作为投票因子
- **架构升级**：实现**分层决策框架**（4层）
  - **L0: 大盘过滤器**（PolicyEvent）✅ 已启用：极度利空（score<-0.5且有重大利空关键词）时暂停选股
  - **L1: 市场状态感知**（Sentiment + MarketRegimeEngine）⚠️ 已关闭：
    - 问题：Sentiment 调用错误（传入单只股票df，应该用全市场指数数据）
    - 待修复后再启用
  - **L2: 权重动态调整**（`get_regime_adjusted_weights`）⚠️ 已关闭：
    - 问题：调节系数（1.2、1.3、0.8等）缺乏历史回测验证，属于"拍脑袋"经验值
    - 问题：系数叠加过猛（熊市1.3 × 恐慌1.3 = 1.69x）
    - 待通过参数扫描（2015-2025年数据）优化系数后再启用
  - **L3: 个股投票**（11策略）✅ 已启用：使用固定的已验证权重进行加权投票
    - 技术面6个: MA, MACD, RSI, BOLL, KDJ, DUAL
    - 基本面3个: PE, PB, PEPB
    - 消息面1个: NEWS
    - 资金面1个: MONEY_FLOW
- **当前实际运行**：L0（PolicyEvent过滤）+ L3（11策略固定权重投票）

## 2026-03-22: DUAL反向 + 投票机制优化（v4）

### 改动内容

**文件**: `src/strategies/ensemble.py`

#### 1. DUAL信号反向处理（第400-416行）
- **原因**：IC分析显示DUAL策略与未来收益强负相关（IC=-0.3877）
- **验证**：反向后IC变为+0.3877（p<0.000001，统计显著）
- **实现**：BUY→SELL, SELL→BUY，confidence和position同步反向
- **效果**：将最差策略转化为最强预测因子

#### 2. 投票机制改进（第555-608行）
- **旧机制问题**：相对比例 `buy_pct = buy_score / total_active` 对权重不敏感
  - 例：3个策略投BUY，权重(1,1,1)和(10,1,1)的buy_pct都接近100%
  - 导致调整权重无法改变决策
  
- **新机制**：净得分 `net_score = (buy_score - sell_score) / active_weight_sum`
  - 权重高的策略对net_score贡献更大
  - 阈值校准为0.07（平衡交易频率和信号质量）
  - 最小票数降为1（避免过度保守，DUAL单票也有效）

#### 3. 保持原始权重（第147-159行）
```python
weights = {
    'BOLL': 1.5, 'MACD': 1.3, 'KDJ': 1.1, 'MA': 1.0,
    'DUAL': 0.9, 'RSI': 0.8, 'PEPB': 0.8, 'PE': 0.6,
    'PB': 0.6, 'NEWS': 0.5, 'MONEY_FLOW': 0.4,
}
```
- **原因**：激进权重方案存在过拟合（样本外恶化-7.11pp）
- **策略**：保守优化，基于v3已验证的稳定权重

#### 4. 关闭动态权重 (`use_dynamic_weights=False`)
- **原因**：避免动态归一化覆盖自定义权重
- **效果**：确保权重配置生效
